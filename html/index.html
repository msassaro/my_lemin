SYNOPSIS

The goal of the LEM-IN project is to find the shortest paths between rooms.
You have a starting point and an ending point.

All ants are starting at the start point and they have to go to the end point. (captain obvious).

For each room (or node), there is only a capacity of 1 ant. So only one ant per room.

TECHNICALS

The lem-in read directly from STDIN or a file sent as a STDIN.

If there is an issue with a line, the recuperation process stops and the lem-in will try to solve
with the informations it has got before the break point.

If there is some issue, the lem-in will quit with ERROR.
